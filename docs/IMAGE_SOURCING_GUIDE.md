## Purpose

This document is the operational companion to `MASTER_PHOTO_LIBRARY.md`. Where that document tells you *what* to source, this document tells you *how* to source it — which platforms to use, in what order, how to search effectively, how to evaluate images for editorial fit, how licensing works across each platform, and how to maintain visual consistency across 44 photographs sourced from potentially different photographers.

## 1. Source Priority Hierarchy

Source images in this order. Move to the next source only when the preceding source yields no suitable result for a given image.

### Priority 1 — Unsplash (unsplash.com)

**Why first:** The Unsplash Licence is perpetual, irrevocable, royalty-free, and covers commercial use with no attribution required (though attribution is encouraged). The contributor base includes many professional wildlife and landscape photographers who have uploaded Africa-quality work. Zero cost. No registration required to download.

**Limitations:**
- No model release guarantee — verify for people images
- Not every subject is well-represented (some niche subjects like mobile camps are sparse)
- Quality is variable — see Section 4 for evaluation criteria

**Licence summary:** Unsplash grants a non-exclusive, worldwide copyright licence to download, copy, modify, distribute, perform, and use photos from Unsplash for free, including for commercial purposes, without permission from or attributing the photographer or Unsplash. The licence does not include the right to compile photos from Unsplash to replicate a similar or competing service.

### Priority 2 — Pexels (pexels.com)

**Why second:** The Pexels Licence is similarly permissive — free for commercial use, no attribution required. The platform aggregates Unsplash content and its own contributor base, so some overlap exists. Useful when Unsplash yields no suitable result.

**Limitations:**
- Smaller Africa-specific photography base than Unsplash
- Same model release caveats as Unsplash

**Licence summary:** All photos on Pexels can be used for free. Attribution is not required. Photos can be modified. Photos can be used for commercial purposes.

### Priority 3 — Wikimedia Commons (commons.wikimedia.org)

**Why third:** Wikimedia Commons hosts an enormous volume of Africa wildlife photography, including many images under Creative Commons licences suitable for commercial use. Quality ranges from professional to amateur; careful filtering required.

**Limitations:**
- Licence varies by image — must check each image individually
- Only use images with CC BY, CC BY-SA, or CC0 (Public Domain) licences
- CC BY requires attribution — must be handled in the site's credits or legal page
- Image resolution is sometimes lower than ideal

**Licences to use:**
- ✅ **CC0 / Public Domain** — no restrictions, no attribution required
- ✅ **CC BY 4.0 / 3.0** — attribution required, commercial use permitted
- ✅ **CC BY-SA 4.0 / 3.0** — attribution required, share-alike (derivative works must use same licence)
- ❌ **CC BY-NC** — non-commercial only; **do not use**
- ❌ **CC BY-ND** — no derivatives; **do not use** (even cropping may constitute a derivative)
- ❌ **All rights reserved** — **do not use**

### Priority 4 — Pixabay (pixabay.com)

**Why last:** The Pixabay Content Licence allows commercial use without attribution. However, image quality for Africa wildlife photography tends to be lower than Unsplash or Pexels. Use only as a fallback when no suitable image is found on the other platforms.

**Licence summary:** All images on Pixabay are released under the Pixabay Content Licence, which allows irreversible, worldwide, non-exclusive use free of charge, including for commercial purposes, without requiring attribution.

## 2. When to Commission Photography Instead

For the following image categories, **stock photography is a significant risk** and commissioned photography is the strongly recommended alternative:

### Team Portraits (IMG-037 through IMG-040)

These images are presented on the About page as named, specific individuals. If stock photography is used:

- The same image may appear elsewhere on the internet associated with a different person or brand
- A visitor who recognises the stock model will immediately undermine brand trust
- There is no way to guarantee the representation matches the person described in the bio

**Recommendation:** Commission a half-day portrait photography session for all four team members simultaneously. A single photographer shooting four subjects in the field is efficient and will produce consistent style across all portraits. If the team is distributed, schedule individual sessions with a briefing document ensuring consistent light and styling.

**If stock must be used for the team portraits:** Search for images with unusual or distinctive styling that are less likely to be reused by other brands. Run a reverse image search (Google Images or TinEye) on any selected portrait before using it to check for prior usage in a professional or brand context.

### Images with Visible People (Model Release)

Any image where a person's face is clearly identifiable carries model release requirements for commercial use. Unsplash and Pexels do not guarantee model releases have been obtained for every image.

**Affected images:** IMG-024 (Maasai elder), IMG-025 (safari guide), IMG-026 (female guide), IMG-028 (family on vehicle), IMG-033 (walking safari guests)

**Mitigation options:**
1. Search for images where identifiable faces are not the focal point (back-of-head, silhouette, figure in landscape)
2. Contact the Unsplash/Pexels photographer directly to confirm model release
3. Source from Getty Images / Adobe Stock where model releases are guaranteed and documented
4. Commission the photography

## 3. Search Strategy

### 3.1 General Principles

**Search in English only.** All major stock platforms index in English regardless of the photographer's nationality.

**Be specific first, broaden if needed.** Start with the most specific query that matches the image brief. Only broaden if no suitable results appear after 2–3 search attempts.

**Use multiple query structures:**
- Subject + Location: `leopard samburu kenya`
- Subject + Mood: `elephant kilimanjaro morning light`
- Subject + Activity: `wildebeest mara river crossing`
- Location + Time of Day: `serengeti golden hour landscape`

**Sort by relevance first, then switch to "latest" if results are thin.** Some subjects have fewer but more recent high-quality contributions.

**Use Unsplash collections.** Many Unsplash photographers curate their Africa work into collections. Searching `africa safari wildlife` and then browsing the Collections tab often surfaces higher-quality curated sets than individual keyword searches.

### 3.3 Difficult Subjects — Alternative Approaches

Some subjects are underrepresented in free stock libraries. For the following images, use these fallback strategies:

**IMG-031 — Mobile Expedition Camp**
Mobile camps are rarely photographed by stock contributors. Search `fly camp africa`, `expedition camp africa plains`, or `lightweight safari camp`. If no suitable result is found, a wide exterior shot of a simple tented camp at the edge of the plains (even a permanent camp) can work if the background is sufficiently open and wild.

**IMG-034 — Bush Plane on Airstrip**
Search `cessna caravan africa airstrip`, `light aircraft africa bush`, `bush plane kenya`. If a direct airstrip shot is unavailable, an aerial shot from inside a light aircraft looking out over the landscape (the plane window and wing visible) can substitute and may be more distinctive.

**IMG-035 — Game Drive POV at Dawn**
This first-person perspective is uncommon in stock photography. Search `inside safari vehicle africa`, `game drive pov africa`, `vehicle bonnet africa dawn`. If a direct POV is not available, a close-up of a safari vehicle dashboard or steering wheel with the landscape visible through the windscreen is an acceptable substitute.

**IMG-036 — Safari Planning Desk**
Search `safari planning materials`, `camp table binoculars map`, `field journal binoculars africa`. If African context is unavailable, a general outdoors planning/adventure desk image (compass, map, hiking context) can work if the warm light and natural materials feel appropriate.

## 4. Image Evaluation Criteria

When reviewing candidate images, evaluate against these criteria in order. Reject any image that fails criteria 1–4; use judgement on 5–8.

### Mandatory Criteria (all must pass)

**1. Technical sharpness.** Zoom to 100% on the primary subject. Eyes (animals, people), fur or feather texture, or leaf detail should be resolved clearly. Motion blur on background is acceptable; blur on the primary subject is not.

**2. Sufficient resolution.** Verify the image meets or exceeds the minimum resolution specified in `MASTER_PHOTO_LIBRARY.md` for the primary slot. For images serving multiple crops (e.g., IMG-001 at 5000px), verify the resolution supports the most demanding crop.

**3. Colour temperature alignment.** The image must integrate with the site's warm earth-tone palette. Hold it mentally against the site's basalt black (`#0C0D0B`), dune gold (`#F0E8D5`), and amber accent (`#D4A017`). Cool-cast images (dominant blue, grey, or green) will clash with the design system. Exceptions: gorilla forest images (intentionally cool-green) and Zanzibar ocean (intentionally turquoise).

**4. No competitor branding.** Scan for visible lodge logos, camp names, vehicle branding (guide company names on vehicle sides), airline livery on aircraft, or identifiable signage that could create brand association issues.

### Quality Criteria (use judgement)

**5. Compositional strength.** Does the image work at the intended aspect ratio? A beautiful image that only works at 1:1 is useless for a 16:9 hero slot. Mentally crop the image to the required dimensions before selecting it.

**6. Editorial authenticity.** Does the image feel earned and real, or staged and performed? Wildlife in natural behaviour. People in genuine context. Landscapes without obvious digital sky replacements or composite manipulation.

**7. Uniqueness.** Avoid images that are already extremely widely circulated and immediately recognisable as stock photography. A reverse image search (Google Images) on any shortlisted candidate will reveal if it has been heavily reused. Prefer images with fewer indexed reuses.

**8. Narrative cohesion.** Will this image make sense alongside the 5–6 other images a user will see on the same page? Variety within consistency. Not every card on the homepage should be shot in the same direction or the same time of day. But they should all feel like they come from the same Africa.

## 5. Editorial Consistency Across 44 Images

The most common failure mode when sourcing photography from multiple contributors is visual fragmentation — the images feel like they came from 44 different shoots, with mismatched colour grades, styles, and moods. This section explains how to mitigate that risk.

### 5.1 The Warm/Natural Test

Before finalising any image, apply this simple test: does the image feel warm and natural? Warmth here means the colour temperature leans amber rather than blue. Natural means the processing feels like the photographer enhanced what was there, not invented something new.

**Pass:** Honey grass, golden hour sidelight, warm dusk shadows, earthy ochre tones, the amber of campfire light.

**Fail:** Hyper-saturated greens, deep artificial blue skies (Clarity/Vibrance pushed past natural), electric white highlights, Hollywood-teal colour grade.

### 5.2 The Scale / Intimacy Balance

The site's editorial identity oscillates between epic (vast landscapes, migration scale, aerial perspective) and intimate (gorilla eye contact, camp candlelight, guide's hands on binoculars). Ensure the final library contains both registers — approximately 50/50.

Images that are always epic: IMG-001, IMG-003, IMG-013, IMG-014 (the crossing), IMG-017, IMG-020.

Images that must be intimate: IMG-015 (gorilla eyes), IMG-016, IMG-024, IMG-025, IMG-029, IMG-030.

If the sourcing session is running heavily epic, prioritise the intimate slots before completion.

### 5.3 Subject Representation

The library should represent the full character of the Rêve Africa offering. Review the completed library against this checklist before finalising:

- [ ] At least 3 images of East African savanna (Kenya/Tanzania)
- [ ] At least 2 gorilla/forest images (Rwanda/Uganda)
- [ ] At least 2 Southern Africa images (Botswana/Namibia)
- [ ] At least 2 images showing people (guides/community)
- [ ] At least 1 aerial image
- [ ] At least 1 camp interior image
- [ ] At least 1 coastal/beach image
- [ ] At least 1 conservation-focused image
- [ ] At least 2 images that will work as portrait (3:4) crops
- [ ] At least 2 wide-span (2×1) gallery images

### 5.4 Light Quality Standards

**Time of day guide:**
- Golden hour (dawn/dusk, ~30 minutes either side of sunrise/sunset): strongly preferred for savanna, wildlife, and landscape
- Midday/harsh light: acceptable only for aerial images and desert (where the bleached quality is intentional)
- Blue hour (just before dawn, just after sunset): excellent for camp interiors and tent glow images
- Overcast: acceptable for forest/gorilla images where dappled light is impossible to predict

### 5.5 Processing Consistency Check

After selecting all images, perform a final side-by-side review. Open all selected images in a grid view (a simple browser tab grid or a tool like macOS Quick Look with multiple files selected). Ask:

- Do these look like they could have been shot by the same editorial team?
- Are there any outliers in colour temperature, saturation, or processing style?
- Are there any two images that are too similar in subject, colour, or composition to appear on the same page?

If an image is an outlier, find a replacement before moving to integration.

## 6. Licensing Administration

### 6.1 Attribution Tracking

Even though Unsplash and Pexels do not require attribution, **recording attribution is strongly recommended** for the following reasons:
- If a licensing dispute arises, documentation of the source proves good-faith use
- The site may wish to add a photo credits page as a goodwill gesture to contributors
- Wikimedia Commons images under CC BY *require* attribution — the tracking system must capture this

**Recommended tracking format (maintain a spreadsheet or table):**

| Image ID | Filename | Platform | Photographer Name | Photo URL | Licence | Attribution Required | Notes |
|----------|----------|----------|-------------------|-----------|---------|----------------------|-------|
| IMG-001 | img-001-maasai-mara-dawn-landscape.jpg | Unsplash | [name] | [url] | Unsplash Licence | No | |
| IMG-024 | img-024-maasai-elder-portrait.jpg | Unsplash | [name] | [url] | Unsplash Licence | No | Model release status: unconfirmed |

### 6.2 Wikimedia Commons Attribution

For any image sourced from Wikimedia Commons under a CC BY licence, attribution **must** appear on the site. The recommended approach:

**Option A (preferred):** Add a `/legal/photo-credits` page listing all CC BY attributed images.

**Option B:** Add attribution in the image's `title` attribute (tooltip on hover) in the HTML.

**Option C:** Add a small credits section in the site footer.

Attribution format: `Photo by [Photographer Name] / Wikimedia Commons / CC BY [version]`

Example: `Photo by Charles J. Sharp / Wikimedia Commons / CC BY-SA 4.0`

### 6.3 Model Release Status

Track model release status for every image containing an identifiable person:

| Image ID | Subject | Model Release | Source | Action if Uncertain |
|----------|---------|---------------|--------|---------------------|
| IMG-024 | Maasai elder | Unknown | Unsplash | Contact photographer or replace with non-identifiable-face image |
| IMG-025 | Safari guide | Unknown | Unsplash | Contact photographer or use back-of-head composition |
| IMG-026 | Female guide | Unknown | Unsplash | Contact photographer or use back-of-head composition |
| IMG-028 | Family | Unknown | Unsplash | Prefer back-of-head/silhouette composition |
| IMG-033 | Walking safari guests | Unknown | Unsplash | Prefer back-of-head/profile composition |
| IMG-037–040 | Team portraits | **Required** | Commissioned | Model release must be signed before use |

**If model release status cannot be confirmed for a stock image showing a person's face in a potentially commercial context:** replace the image with one where the face is not clearly identifiable (silhouette, back-of-head, in-context environmental portrait at distance).

### 6.4 Paid Stock as a Fallback

For images that cannot be adequately sourced from free platforms — particularly IMG-014 (wildebeest crossing), IMG-015 (gorilla), and the team portraits — Adobe Stock or Getty Images are the recommended paid alternatives.

**Adobe Stock:** Integrated with Creative Cloud. Single-image licences are available from approximately USD $29.99. All images include model and property release documentation where applicable.

**Getty Images:** Higher-cost but the largest and best-documented Africa wildlife collection. Suitable for any image that must be guaranteed model-released or that represents a specific, named location.

## 7. File Handling After Sourcing

### 7.1 Download Format

Always download the highest-resolution version available from the source platform. Do not download resized or compressed versions.

**Expected file types:** JPEG (most common from stock platforms). If a TIFF or PNG is available at higher quality, download that instead and convert to JPEG for delivery.

### 7.2 File Organisation Before Delivery

Organise sourced images in this folder structure before handing to the developer:

```
sourced-images/
├── raw/                    ← Original downloads, unmodified, named by platform ID
│   ├── unsplash-abc123.jpg
│   └── ...
├── renamed/                ← Renamed to production filenames per this guide
│   ├── img-001-maasai-mara-dawn-landscape.jpg
│   ├── img-002-serengeti-plains-landscape.jpg
│   └── ...
└── ATTRIBUTION_LOG.md      ← Tracking spreadsheet or markdown table
```

The `raw/` folder is an archive. The `renamed/` folder is what gets handed to the developer for integration.

### 7.3 No Editing Required (by the sourcing person)

The developer will handle all format conversion (WebP/AVIF), responsive sizing, and optimisation via `next/image`. The sourcing task should deliver original, unmodified, full-resolution JPEGs. **Do not resize, compress, or watermark images before delivery.**

The only exception is if a specific crop is needed to remove an unwanted element (e.g., a visible logo) — in that case, note the crop in the attribution log and deliver the cropped version alongside the original.

### Per-Image Sourcing Checklist

For each image:
- [ ] Search primary platform (Unsplash) with 2–3 query variations
- [ ] Shortlist 3–5 candidates
- [ ] Apply mandatory evaluation criteria (sharpness, resolution, colour, no branding)
- [ ] Select the strongest candidate
- [ ] Run reverse image search on selected image (Google Images / TinEye)
- [ ] Note photographer name, photo URL, and licence in attribution log
- [ ] Note model release status if the image contains people
- [ ] Download highest-resolution version
- [ ] Save to `sourced-images/raw/` with original platform filename
- [ ] Copy to `sourced-images/renamed/` with production filename from this guide
